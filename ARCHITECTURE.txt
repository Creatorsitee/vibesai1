SYSTEM CHARACTERISTICS
─────────────────────────────────────────────────────────────────────────────

Auto-Detection Algorithm:
  1. Check if GeminiService is initialized
  2. Check if ClaudeService is initialized
  3. If Claude available → use Claude as primary (better for code)
  4. Else if Gemini available → use Gemini as primary (fastest)
  5. Else → show "No providers configured" error

Fallback Mechanism:
  1. User selects unavailable provider → error shown
  2. Provider becomes unavailable during use → auto-switch to another
  3. Last provider in list is fallback
  4. If all providers down → show error notification

Retry Strategy (Both Services):
  • Max Retries: 3
  • Initial Delay: 1000ms
  • Max Delay: 10,000ms
  • Backoff Multiplier: 2x
  • Total Max Time: ~13 seconds per request

Security:
  • API keys never logged except in error context
  • Keys treated as sensitive environment variables
  • No storage in localStorage
  • No transmission to third-party services
  • Secure HTTPS for all API calls

Performance:
  • Streaming responses for real-time feedback
  • Debounced UI updates
  • Memoized components to prevent re-renders
  • Efficient error handling
  • Minimal bundle size overhead (~20KB)
